# 競技遊戲對戰紀錄系統 - PRD (依據 PRD Skill 產出)

## 1. 專案概述

### 背景與動機
在許多含有卡牌競技元素的遊戲（例如皇室戰爭 Clash Royale、卡牌對戰或其他含有配置/流派的連線對戰遊戲）中，玩家雖然擁有習慣使用的「主戰牌組」或「主要配置」，但常會遇到被某些特定卡牌或流派克制的情況。如果單憑記憶，難以精確得知針對特定陣容的實際勝率。因此，玩家需要一個系統化的對戰紀錄工具，以客觀數據來釐清自己的勝負輪廓。

### 目標用戶
- 競技遊戲的中重度玩家
- 喜歡鑽研卡牌陣容剋制關係、希望透過覆盤來精進實力與改善勝率的玩家
- 需要管理多套主要配置（主戰牌組）的進階玩家

### 核心價值主張
幫助玩家用最簡單直觀的方式紀錄對局結果，透過數據點出「自己最怕什麼牌」及「最穩吃什麼牌」，進而提供調整策略或針對性訓練的客觀依據。

## 2. 功能需求

以下列出本系統的主要功能與對應的使用者故事 (User Stories)：

1. **主戰牌組管理**
   - 作為 **玩家**，我希望 **可以新增並儲存我的多套常用主戰牌組（如：電磁炮狂暴、石頭人）**，以便 **之後紀錄對戰時能快速選用，並能分別追蹤每套牌的表現**。
2. **對戰紀錄輸入**
   - 作為 **玩家**，我希望 **能在每局遊戲後紀錄我方牌組、對手使用的特定關鍵卡牌（如：火箭、皇家快遞）以及勝負結果**，以便 **累積數據庫**。
3. **對戰數據分析**
   - 作為 **玩家**，我希望 **可以看到我的特定主戰牌組面對不同的對手單卡時的勝/負場數與勝率統計**，以便 **了解這套牌組被什麼卡克制**。
4. **歷史紀錄檢視與編輯**
   - 作為 **玩家**，我希望 **可以查看過去所有的紀錄列表，並具備修改與刪除的功能**，以便 **在紀錄錯誤時能夠即時修正，保持數據準確**。
5. **對戰備註與檢討**
   - 作為 **玩家**，我希望 **可以在每筆對戰紀錄中填寫反思與備註內容**，以便 **隨時回顧當下的戰術失誤或對手特點**。

## 3. 非功能需求

- **技術限制**：
  - 後端：Python + **Flask** 框架
  - 前端渲染：**Jinja2** 模板系統 (可搭配 Bootstrap、Tailwind 或純 CSS)
  - 資料庫：**SQLite** (輕量關聯式資料庫，便於開發與部署)
- **效能與體驗考量**：
  - 因為玩家常在遊戲間的小空檔紀錄勝負，操作流程需盡量簡化，UI 操作以簡單的點擊和選單為主。
  - 建議具備基礎響應式網頁設計 (RWD)，以利行動裝置上使用。
- **安全考量**：
  - 基礎表單防護 (如防範 SQL Injection；Flask 搭配 ORM 較易處理)。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 範圍層級 (MoSCoW) | 功能項目與說明 |
| :--- | :--- | :--- |
| **高** | **Must Have** | - 建立 / 刪除主戰牌組名稱<br>- 新增單局對戰紀錄（選擇我方牌組、輸入對手關鍵卡牌名稱、選擇勝負）<br>- 基本歷史對戰清單頁面<br>- 特定卡牌 / 牌組對抗勝率總計（純文字/表格顯示） |
| **中** | **Should Have** | - 編輯已存在的對戰紀錄<br>- 為對戰加入自訂備註文字<br>- 主戰牌組更新/修改功能 |
| **低** | **Nice to Have** | - 圖表視覺化呈現（如勝率圓餅圖）<br>- 使用者帳號註冊/登入（提供多使用者區隔）<br>- 卡牌的自動完成/預設常用卡牌清單 (Autocomplete) |
